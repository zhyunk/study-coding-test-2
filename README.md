## 📌 규칙

### 1. 풀이 방식
- 하나의 문제를 다양한 방식으로 풀어본다.
  - 예를 들면, stream을 사용한 코드와 사용하지 않은 코드를 모두 작성한다.

### 2. 기록 관리
- 코드 파일 디렉토리의 상위에 있는 `README.md`에 풀이 완료 날짜를 추가한다.

### 3. 코드 스타일
- 첫 줄에 주석으로 문제 링크와 내용을 추가한다.
  - 링크는 복붙하기 쉽도록 앞에 아무 문자도 없어야 한다.
    > ```kotlin
    > /**
    > https://url.url
    >  * 문제 설명
    >  * ...
    >  */
    > ```
- 모든 코드 파일의 마지막에는 빈 줄을 1개 이상 추가한다.
